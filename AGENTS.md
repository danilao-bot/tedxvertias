# TEDxVeritasUniversity — Agent Guide

## Project Overview

This repository contains the website for **TEDxVeritasUniversity**, a TEDx event hosted at Veritas University, Abuja, Nigeria. The event theme is "Neo-Intelligence: Shaping an Intelligent Africa Through Innovation and Culture" and is scheduled for Saturday, 16th May 2026.

The codebase is split into **two parallel versions** of the same site:

1. **`tedxvertias-next/`** — The primary modern version. A Next.js 16 + React 19 application using TypeScript and Tailwind CSS v4.
2. **Root-level HTML files** (`index.html`, `register.html`, `email.html`) — Static fallback/legacy versions that mirror the Next.js pages. These are standalone files with no build step.

**Important:** Changes to content, styling, or functionality should ideally be applied to **both** the Next.js app and the root HTML files to keep them in sync. The root HTML files are frequently used as fallbacks or for quick hosting.

## Build and Run Commands

### Next.js Frontend

All commands run from `tedxvertias-next/`:

```bash
# Development server (Turbopack, port 3000)
npm run dev

# Production build
npm run build

# Start production server
npm run start

# Lint check
npm run lint
```

### Express Backend

All commands run from `tedxvertias-next/server/`:

```bash
# Install server dependencies
npm install

# Start server (port 5000)
npm start
# or
npm run dev
```

The server must be running for the `/admin` page to function (it fetches from `http://localhost:5000/api`).

## Runtime Architecture

### Frontend (Next.js)
- Uses the **App Router** (`app/` directory).
- **`layout.tsx`** provides the root HTML shell, loads Google Fonts and Font Awesome via `<head>`, and sets page metadata.
- **Client components** use the `'use client'` directive. These include:
  - `page.tsx` (landing)
  - `admin/page.tsx`
  - `register/page.tsx`
  - `email/EmailContent.tsx`
  - All files in `components/`
- **Path alias:** `@/*` resolves to `./*` (project root relative to `tedxvertias-next/`).

### Backend (Express)
- Standalone Node.js server, **not** a Next.js API route.
- Runs on port `5000` by default (configurable via `PORT` env var).
- CORS is enabled globally.
- Routes are prefixed with `/api`:
  - `GET  /api/register` — List all registrations
  - `GET  /api/admin/registrations?status=&search=` — Filtered registrations
  - `GET  /api/admin/stats` — Registration statistics
  - `DELETE /api/admin/registrations/:id` — Delete a registration
  - `POST /api/certificate/send` — Send certificate to one user
  - `POST /api/certificate/send-all` — Batch send certificates
  - `GET  /api/certificate/status` — Batch job progress
  - `POST /api/certificate/reset` — Reset batch status

### Database (Supabase)
- Table: `registrations`
- Columns (inferred): `id`, `full_name`, `email`, `phone`, `status` (UnderGraduate | PostGraduate | Staff | Guest | Alumni), `department`, `created_at`, `certificate_sent` (boolean), `certificate_sent_at`

## Code Style Guidelines

- **TypeScript:** Strict mode is enabled. Use explicit types where possible.
- **CSS:** A mix of Tailwind utility classes and inline `style` props. The landing page (`globals.css`) contains extensive custom CSS for animations, responsive breakpoints, and the dark theme.
- **CSS Variables:** Defined in `:root` inside `globals.css` (and duplicated in the static HTML files). Key brand colors:
  - `--tedx-red: #EB0028`
  - `--veritas-green: #007A33`
  - `--bg-primary: #000000`
  - `--text-secondary: #9A9A9A`
- **Components:** Keep components in `components/`. Reusable UI pieces (Navbar, Cursor, Countdown) are separated from page-specific content.
- **Fonts:** Montserrat (headings), Space Grotesk (body), Crimson Pro (italic/taglines).
- **Responsive:** Mobile-first with breakpoints at `1024px`, `768px`, `428px`, `375px`, and `320px`.

## Testing Instructions

There are **no automated tests** in this project. All testing is manual:

1. **Landing Page:** Check scroll-reveal animations, canvas animation performance, navbar scroll behavior, mobile hamburger menu, and countdown accuracy.
2. **Registration:** Submit the form and verify the row appears in Supabase. Check conditional department field (hidden for Guest/Alumni).
3. **Email Confirmation:** After registration, confirm redirect to `/email` and test the "Sync to Calendar" button (iOS should download `.ics`, others open Google Calendar).
4. **Admin Dashboard:**
   - Navigate to `/admin`.
   - Log in with admin credentials (validated against server env vars).
   - Verify stats load, filtering works, search works, and the "Send" / "Send All Pending" buttons trigger certificate emails.
5. **Static HTML Files:** Open `index.html`, `register.html`, and `email.html` directly in a browser to ensure they render identically to the Next.js versions.

## Security Considerations

⚠️ **Known security issues agents should be aware of:**

1. **Exposed Supabase Credentials:** The Supabase anonymous key and URL are hardcoded in:
   - `tedxvertias-next/app/register/page.tsx`
   - `register.html`
   - This is an **anon** role key, but it still allows public insert/select on the `registrations` table depending on RLS policies.

2. **Admin Authentication:** The admin page uses **Basic Auth via request headers** (`email` + `password` sent as plain headers on every fetch). There are no secure sessions, JWTs, or cookies. Credentials are compared against server environment variables.

3. **No Input Sanitization:** Form inputs are sent directly to Supabase. The backend does not perform additional validation beyond what the frontend enforces.

4. **Local Development Only:** The admin dashboard is hardcoded to call `http://localhost:5000`. It will not work in production without updating `API_URL`.

5. **Certificate Batch Rate Limiting:** The certificate sender adds a 60-second delay between emails to avoid Resend rate limits, but there is no queue or retry mechanism beyond simple logging.

## Deployment Notes

- The Next.js app is designed for **Vercel** deployment (standard `create-next-app` setup).
- The Express server is **not** integrated into the Next.js runtime. If deploying to Vercel, the server must be hosted separately (e.g., on a VPS, Railway, Render, etc.) or converted to Next.js API routes.
- The root static HTML files can be served from any static host (GitHub Pages, Netlify, etc.) without a build step.

## Agent-Specific Notes

- **Next.js Version Warning:** This project uses Next.js 16, which may have APIs and conventions that differ from your training data. If unsure, check the local docs in `node_modules/next/dist/docs/` or the official Next.js 16 documentation.
- **Static HTML Parity:** When updating content (event dates, speaker info, schedule, copy), remember to update both the Next.js app and the root HTML files.
- **Canvas Animation:** `HeroCanvas.tsx` is a performance-sensitive component using `requestAnimationFrame`. Be careful when modifying particle counts or connection logic.
- **Event Date:** The countdown and calendar sync target **May 16, 2026**. Updating this requires changes in:
  - `Countdown.tsx`
  - `page.tsx` (hero date)
  - `email/EmailContent.tsx`
  - `index.html`
  - `register.html`
  - `email.html`
  - `server/utils/pdfGenerator.js`
